## 1. Purpose

This document defines security training requirements for all GoFarther AI personnel and provides a log for tracking training completion, as required by SOC 2 Trust Service Criteria (CC1.4 — Security Awareness).

## 2. Scope

All individuals with access to GoFarther AI systems:
- Full-time employees
- Part-time employees
- Contractors and consultants
- Third-party service providers with system access

## 3. Required Training Topics

### 3.1 Core Security Training (All Personnel)

| # | Topic | Description | Frequency |
|---|-------|------------|-----------|
| 1 | **Information Security Policy Overview** | Company security policies, data classification, acceptable use | Annually |
| 2 | **Password Security** | Strong passwords, password managers, no password sharing | Annually |
| 3 | **Phishing Awareness** | Identifying phishing emails, social engineering tactics, reporting procedures | Annually |
| 4 | **Data Handling** | Proper handling of confidential/internal data, encryption requirements | Annually |
| 5 | **Incident Reporting** | How to identify and report security incidents, escalation procedures | Annually |
| 6 | **Physical Security** | Device security, screen locking, full-disk encryption | Annually |
| 7 | **Remote Work Security** | VPN usage, public Wi-Fi risks, secure home office practices | Annually |

### 3.2 Technical Security Training (Engineering Team)

| # | Topic | Description | Frequency |
|---|-------|------------|-----------|
| 8 | **Secure Coding Practices** | OWASP Top 10, input validation, output encoding, parameterized queries | Annually |
| 9 | **Secrets Management** | Environment variables, no secrets in code, key rotation procedures | Annually |
| 10 | **Code Review Security** | Reviewing for security vulnerabilities, common patterns to flag | Annually |
| 11 | **Dependency Security** | Monitoring for CVEs, update procedures, supply chain risks | Annually |
| 12 | **Logging Best Practices** | What to log, what NOT to log (PII, secrets), audit trail requirements | Annually |
| 13 | **Incident Response for Engineers** | Containment procedures, rollback, key rotation, evidence preservation | Annually |

### 3.3 New Hire Training

All new hires must complete the following within their first week:

- [ ] Core Security Training (topics 1-7)
- [ ] Read and acknowledge Information Security Policy
- [ ] Read and acknowledge Access Control Policy
- [ ] Set up MFA on all company accounts
- [ ] Enable full-disk encryption on all devices
- [ ] Technical training (topics 8-13) if engineering role
- [ ] Sign security policy acknowledgment form

## 7. Training Effectiveness

### 7.1 Metrics

| Metric | Target |
|--------|--------|
| Training completion rate | 100% of active personnel |
| New hire training within first week | 100% |
| Phishing simulation click rate | < 10% |
| Security incident caused by trained personnel | 0 |

### 7.2 Assessment

- Training effectiveness reviewed annually
- Phishing simulation results tracked quarterly
- Additional training assigned for personnel who fail phishing tests
- Training program updated based on:
  - New threat landscape
  - Post-incident findings
  - Industry best practices
  - Employee feedback
